Today `scripts/factory.sh` always emits a Python stub (`<slug>.py`). Teams on other stacks have to hand-edit the generated file.

**Proposal:** add a `STUB_LANG` env var (default `py`) that selects the stub file extension and body — e.g. `py`, `js`, `ts`, `go` — while keeping the deterministic slug → filename / function-name derivation.

- `STUB_LANG=js` produces `<slug>.js` with an exported function stub
- Default behavior (no `STUB_LANG`) stays Python, unchanged
- `docs/factory.md` documents the new option
