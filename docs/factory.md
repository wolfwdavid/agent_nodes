# AI-free templated factory

Turn a GitHub issue into a deterministic **spec + plan + code stub** on a branch, with a pull
request opened automatically. No LLM, no API keys, no cost — it runs on GitHub-hosted runners
using the built-in `GITHUB_TOKEN`.

## What it produces

For issue `#N` titled *"My feature"*, it creates a branch `factory/issue-<N>-<slug>` containing
`factory/issue-<N>-<slug>/`:

| File | Contents |
|---|---|
| `SPEC.md` | Title, source-issue metadata, the issue body folded in as the summary, and an acceptance-criteria checklist |
| `PLAN.md` | A task checklist referencing the stub and the issue |
| `<slug>.py` | A Python stub with a function named after the issue, raising `NotImplementedError` |

…then opens a PR titled `Factory: #N <title>` that refs the issue.

Everything is generated by string templating in [`scripts/factory.sh`](../scripts/factory.sh) — the
output is fully deterministic and predictable.

## How to run it

**1. Label an issue (hands-off)**
Add the **`factory`** label to any issue. The workflow fires on the label event and opens a PR.

**2. Manual dispatch**
GitHub → **Actions** tab → **AI-free templated factory** → **Run workflow** → enter the issue number.

**3. Locally**
```bash
bash scripts/factory.sh <issue-number>
```
Requires `git` and an authenticated [`gh`](https://cli.github.com/) CLI.

## Requirements

For the automated paths (1 and 2) to open PRs, the repo needs:

> **Settings → Actions → General → Workflow permissions** →
> **"Read and write permissions"** + **"Allow GitHub Actions to create and approve pull requests."**

(Already enabled on this repo. The local path #3 needs no settings.)

## Files

- [`scripts/factory.sh`](../scripts/factory.sh) — the generator (templates are inline; edit here to customize).
- [`.github/workflows/factory.yml`](../.github/workflows/factory.yml) — triggers on the `factory` label or manual dispatch.

## Customizing the templates

Open `scripts/factory.sh` and edit the heredocs for `SPEC.md`, `PLAN.md`, and the code stub.
To change the stub language, swap the `.py` filename and the heredoc body — the slug/function-name
derivation works for any language.
